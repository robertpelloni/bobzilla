# VISION

## The Ultimate Goal
**Bobzilla: The browser that doesn't suck.**
Bobzilla is designed to be the Internet's last line of defense against the Chromium monoculture. It combines the privacy DNA of Firefox with Chromium-competitive performance, robust DRM pragmatism for real-world usability, and an absolute zero-surveillance baseline. It is not just a document viewer; it is an autonomous "Agent Shell" tailored for maximum user satisfaction.

## Core Foundational Concepts
1. **Privacy by Default:** No opt-outs, no telemetry, no Mozilla accounts, no Pocket, no sponsored tiles. Privacy is the uncompromisable default state.
2. **Performance Matters:** We aim to close the gap with V8. SpiderMonkey must be aggressively optimized, exploring hybrid or alternative engine architectures if necessary to deliver Chromium-level speed.
3. **DRM Pragmatism:** We despise DRM, but users need Netflix and streaming services. We ensure maximum achievable DRM support (Widevine L3 and beyond, if possible) to make Bobzilla a viable daily driver.
4. **Permanent Manifest V2 Support:** Manifest V2 and the full webRequest API will be preserved forever. Ad blocking must remain uncompromised and superior.
5. **Absurdist & Autonomous Branding:** We embrace the identity of Bobzilla. The browser operates as a gateway to the "Bob" ecosystem (Bobcoin, Bobfilez), shifting towards an autonomous agent model that integrates native voice navigation and decentralized web features.

## User-Satisfaction Design
- **Frictionless Onboarding:** A clean, minimal default theme with immediate access to advanced features like vertical tabs and workspaces.
- **Uncompromised Usability:** The browser should simply work for modern web apps and media, without forcing the user to tinker extensively to fix broken streaming or slow load times.
- **Empowerment over Extraction:** The browser works exclusively for the user. It does not harvest data to sell, nor does it serve ads natively.
